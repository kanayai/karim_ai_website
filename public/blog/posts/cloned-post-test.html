<h1>The Data Scientist's Dilemma: OneDrive vs. GitHub</h1>

<p>As Data Scientists using <strong>Mac, Windows, or Linux</strong>, we live in two worlds. We deal with massive datasets (perfect for Cloud Storage) and version-dependent code (perfect for Git). </p>

<p>Deciding where to store a project can be confusing. Make the wrong choice, and you end up with corrupted repositories or lost versions. This guide breaks down exactly where your files belong.</p>

<h2>1. The Decision Matrix: What Goes Where?</h2>

<p>The fundamental difference is simple: <strong>OneDrive syncs the "Now"</strong> (making folders identical across devices), while <strong>GitHub tracks the "History"</strong> (how a file evolved line-by-line).</p>

<div class="callout callout-tip">
    <strong>üß™ The "Notepad Test" (Rule of Thumb)</strong><br>
    Open your file in a basic text editor (Notepad/TextEdit).
    <ul>
        <li>Can you read it? (Python script, Markdown, CSV) &rarr; <strong>Use GitHub.</strong></li>
        <li>Does it look like alien gibberish? (PDF, Excel, .pkl, .exe) &rarr; <strong>Use OneDrive.</strong></li>
    </ul>
</div>

<h2>2. The "Mirrored" Workflow (Recommended)</h2>

<p>For research and data science, you shouldn't choose one or the other. <strong>You need both, but they must remain separate.</strong></p>

<h2>3. The Danger Zone: Git INSIDE OneDrive</h2>

<div class="callout callout-danger">
    <strong>‚ö†Ô∏è CRITICAL WARNING</strong><br>
    Never initialize a Git repository <em>inside</em> a synced OneDrive/Dropbox/iCloud folder.
</div>

<p>Why? Sync clients (OneDrive) and Version Control (Git) fight over the hidden <code>.git</code> folder. This leads to:</p>
<ul>
    <li><strong>Repo Corruption:</strong> OneDrive may "sync" a file lock that Git needs.</li>
    <li><strong>Conflicted Copies:</strong> You will end up with files named <code>script.py (Computer Name Conflicted Copy)</code>.</li>
</ul>

<h2>4. How to Fix It (Moving Out of OneDrive)</h2>

<p>If your project is currently stuck in OneDrive, follow these steps. <strong>Do NOT move it to "Documents"</strong> (as Documents often auto-syncs to Cloud too).</p>

<h3>Step 0: Create a Safe "Projects" Folder</h3>
<div class="os-grid">
    <div class="os-card">
        <strong>üçé macOS</strong>
        Create <code>/Users/name/Projects</code><br>
        <small>(Do not put in Documents!)</small>
    </div>
    <div class="os-card">
        <strong>ü™ü Windows</strong>
        Create <code>C:\Users\name\Projects</code><br>
        <small>(Do not put in Documents!)</small>
    </div>
    <div class="os-card">
        <strong>üêß Linux</strong>
        Create <code>/home/name/Projects</code><br>
        <small>(Safe anywhere in ~)</small>
    </div>
</div>

<h3>Step 1: On Your Primary Computer</h3>
<ol>
    <li><strong>Move the Folder:</strong> Drag your project folder <strong>OUT</strong> of OneDrive and into your new <code>Projects</code> folder.</li>
    <li><strong>Verify:</strong> Open your terminal (Git Bash on Windows).
        <pre><code>cd ~/Projects/my-project
git status 
    </li>
    <li><strong>Cleanup:</strong> Check OneDrive. If a copy remains there, delete it to stop the sync confusion.</li>
</ol>

<h3>Step 2: On Your Secondary Computer</h3>
<p>Do not try to "fix" the existing folder on your second computer. It's faster to start fresh.</p>
<ol>
    <li><strong>Let OneDrive Sync:</strong> Wait for OneDrive to update. It should automatically remove the old folder.</li>
    <li><strong>Clone Fresh:</strong> Download a clean connection to GitHub in your safe local folder.
        <pre><code>cd ~/Projects
git clone https://github.com/yourname/my-project.git 
    </li>
</ol>

<h2>Summary</h2>
<ul>
    <li><strong>Code</strong> lives in <strong>~/Projects</strong> (Git).</li>
    <li><strong>Data</strong> lives in <strong>~/OneDrive</strong> (Cloud).</li>
    <li><strong>Never</strong> put Git inside OneDrive or Documents (if synced).</li>
</ul>
